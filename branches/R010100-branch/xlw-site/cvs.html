About SourceForge.net, Open Source, CVS, etc.

XLW's CVS Repository

If you're new to CVS please read the SourceForge.net CVS documentation, especially their CVS FAQ. For general (and more advanced) information consult the free CVS Book and the CVS Docs at www.loria.fr

You can browse the CVS tree: this gives you a great view into the current status of the project's code. You may also view the complete histories of any file in the repository.

Anonymous CVS Access

The modules can be checked out through anonymous (pserver) CVS with the following instruction set. The module you wish to check out must be specified as the modulename. When prompted for a password for anonymous, simply press the Enter key.

Unix:

export CVSROOT=:pserver:anonymous@cvs.xlw.sf.net:/cvsroot/xlw
cvs login
cvs -z3 co -P xlw

Win32:

SET CVSROOT=:pserver:anonymous@cvs.xlw.sf.net:/cvsroot/xlw
cvs login
cvs -z3 co -P xlw

After initial checkout, you can execute cvs commands in this directory without the CVSROOT exporting/setting. For example:

cvs update

If you are behind a firewall ask your sysadmin to open the cvspserver port (2401) for outgoing connection from your machine to cvs.xlw.sf.net


Developer CVS Access via SSH

If you are a SourceForge.net-recognized XLW developer you can access the CVS tree via SSH. SSH1 must be installed on your client machine. Substitute modulename and developername with the proper values. Enter your site password when prompted.

Unix:

export CVSROOT=developername@cvs.xlw.sf.net:/cvsroot/xlw
export CVS_RSH=ssh
cvs -z3 co -P xlw

Win32:

SET CVSROOT=developername@cvs.xlw.sf.net:/cvsroot/xlw
SET CVS_RSH=ssh
cvs -z3 co -P xlw

After initial checkout, you can change into this directory and execute cvs commands without the CVSROOT exporting/setting. For example:

cvs update
cvs commit -m "comments for this commit"
cvs add myfile.c

If you are behind a firewall you need to open the shh port (22) for outgoing connection from your machine to xlw.sf.net, shell.sf.net, and upload.sf.net (the latter just in case you plan to help the XLW release process).
